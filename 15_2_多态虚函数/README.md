**虚函数**  
目的: 对于不同对象内容(而非指针类型)，执行不同的行为  
往往是基类virtual,派生类各自实现同名函数 ;; 传递不同派生类对象调用的display函数不同  

**virtual析构函数**     
目的:避免内存泄漏 特别是类内有申请的动态内存，需要清理和释放的时候   
new delete p时候，不再根据p的指针类型调用析构函数,而是根据p指向的对象内容决定调用对应类的析构函数   

**pureVirtual**    
抽象类只起到提供统一接口的作用   
纯虚函数 virtual void func()=0;  有派生类进行实现   
purevirtual不存在于虚函数表当中   

**multiDerivedVirtual**  
对象的存储布局  
多继承\存在虚函数情况下 类class的占据size大小  
虚函数的入口地址 = 8bytes (64位)   

**说明**  
1.	虚拟继承 class A:virtual class B 是为了解决多基派生时的存储二义性
	虚函数 实现了多态性
2.	只有虚函数才会被子类覆盖(函数名 返回值 参数个数和类型相同)  ―― override
3.	回避虚函数――域运算符: 往往是类的成员函数(或者友元)
	例如: baseobj->Base::display();  //强制使用基类中定义的函数版本

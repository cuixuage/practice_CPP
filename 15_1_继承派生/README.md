**基类不能被继承成员**   
1.基类构造函数\析构函数  
2.基类重载的运算符 例如:"="  
3.友元关系  

**权限**  
1.无论以何种方式继承,子类都不能直接访问基类private私有变量  
注: 如果想对子类开放接口,基类可以考虑将数据成员、成员函数放入protected   
2.子类内部 能够访问除了基类private以外的 数据\函数   
3.子类对象 外部只能访问pulic继承基类的public成员  


**单基派生**     
往往: 子类构造函数中显示调用基类的构造函数    
多基派生:    
子类初始化列表中     
派生类名(总参数表):基类名1(参数表1)，基类名2(参数表2)，……，基类名N(参数表N)  
{  
	//函数体  
}  
虚基类构造函数的调用总是先于普通基类的构造函数   

**派生二义性**
1.成员名二义性  
子类继承自多基类 基类中有相同的成员名  
解决: 域限定符   
2.存储二义性   
某基类在派生类的派生类(孙子)中被多重拷贝   
解决: 儿子类 :虚继承(virtual关键字) 基类。使得孙子类中不出现存储二义性   

总结: 先构造则先析构
